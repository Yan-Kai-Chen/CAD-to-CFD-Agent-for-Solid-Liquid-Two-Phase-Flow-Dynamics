# Dewaxing Agent Case Study

This directory is the documentation home for the complex solid-liquid dewaxing
application.

The dewaxing case is now positioned after the compact Agent benchmark ladder:

```text
benchmark ladder -> dewaxing application -> FastFluent-to-Fluent guidance
```

## Working Title

```text
An Agent Framework for FastFluent-to-Fluent Simulation of Complex Solid-Liquid Dynamics
```

## Current Core Evidence

- FastFluent native study: `15` runs.
- FastFluent native validation: `10` cases and `20,525,400` cell-time steps.
- Agent iteration: `46` native runs and `80,140,200` cell-time steps.
- Accepted candidate: `path106_initial6`.
- Completed Fluent retrospective confirmation:
  - early shock `0-4.1 s`;
  - risk window near `100.7 s`;
  - full melt near `409.0 s`.

## Public Asset Mapping

The dewaxing case study uses public assets in four roles:

| role | public assets | purpose |
| --- | --- | --- |
| Implementation | `src/fromcad2cfd_fastcfd/dewaxing_*.py` | Generate native reduced-order evidence, validation packs, Agent iteration records, paper evidence, and Fluent guidance. |
| Public fixture | `examples/postprocessing/dewaxing_result_pack/` | Validate the result-pack contract without private Fluent case/data files. |
| Post-processing contract | `src/fromcad2cfd_postprocessing/dewaxing_result_pack.py` | Check public-safe dewaxing result-pack structure. |
| Tests | `tests/unit/test_fastcfd_dewaxing_*.py` | Preserve the public behavior and claim boundaries. |

Large local `sandbox/output/*_real` evidence packs are not required public
inputs. They can be referenced as local evidence, but public commands should
use generated outputs or sanitized fixtures.

## Claim Boundary

FastFluent is the Agent's reduced-order guidance and screening layer. Completed
Fluent evidence is used for retrospective confirmation. The workflow does not
claim that FastFluent replaces final Fluent validation.

See `docs/public_asset_framework_map.md` for the repository-wide rule that
keeps original public assets in their engineering modules while this directory
serves as the paper-facing case-study route.
